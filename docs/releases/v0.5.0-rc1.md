# workflow_bundle v0.5.0-rc1 Release Notes

发布日期：2026-04-18

对应 tag：`v0.5.0-rc1`

## 概览

`v0.5.0-rc1` 是 `workflow_bundle` 首个采用语义化版本统一治理的预发布版本。
这一版的目标不是引入新的论文业务能力，而是把公共仓库定位、版本号治理和发布识别能力先收口稳定，为后续正式版打底。

## 本次重点

### 1. 建立正式版本治理

- 新增根目录 `VERSION`，作为 bundle 当前版本的单一真源。
- 新增 `CHANGELOG.md`，用于维护结构化版本变更记录。
- 新增 `python3 workflow_bundle/tools/cli.py version` 与 `--version`，可直接查询当前 semver、建议 tag、commit 与 dirty 状态。
- 新增 `workflow/11-versioning-and-release.md`，明确 semver、tag 和最小发版流程。

### 2. 让运行产物可反查 workflow 版本

- `resume` / handoff 现在会显示当前 bundle version。
- `build_summary.json`、`release_summary.json`、`final_summary.json`、`selftest_summary.json` 现在都会记录生成它们时的 bundle 版本信息。
- 后续出现“这个 DOCX 是由哪一版 workflow 生成的”这类问题时，可以直接从机器可读摘要回查。

### 3. 清理公共仓库中的项目残留

- 公开说明文档已去项目化，不再把仓库描述成某个具体论文项目的运行快照。
- `workflow/configs/current_project_manifest.json` 与 `workflow/configs/current_workspace.json` 已收口为官方示例实例。
- `tools/core/extract.py` 中的环境描述文案已去除项目硬编码输出，改为按当前技术栈生成中性说明。

## 对使用者的直接影响

- 现在可以明确回答“当前 workflow 是什么版本”，不再依赖零散中文 milestone tag 猜测。
- 新对话接手 workflow 时，推荐先运行 `python3 workflow_bundle/tools/cli.py version` 与 `python3 workflow_bundle/tools/cli.py resume`。
- 中文 tag 仍可继续作为阶段性里程碑保留，但正式版本请优先认 semver tag。

## 回归结论

本次预发布前已完成以下校验：

- `python3 -m py_compile tools/cli.py tools/core/bundle_version.py tools/core/release_summary.py tools/core/runtime_state.py tools/core/selftest.py`
- `bash workflow/scripts/check_bundle_sync.sh`
- `python3 tools/cli.py selftest`
- `git diff --check`

## 已知边界

- 这是预发布版本，不是正式版 `v0.5.0`。
- 当前版本主要解决的是版本治理、公共仓库清理与发布识别问题，不代表工作流功能面已经停止演进。
- 如需进入正式版，下一步应重点确认 README、CHANGELOG、release note 与 tag 口径保持一致，再决定是否升级到 `v0.5.0`。
