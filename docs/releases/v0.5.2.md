# workflow_bundle v0.5.2 Release Notes

发布日期：2026-04-18

对应 tag：`v0.5.2`

## 概览

`v0.5.2` 是基于 `v0.5.1` 的补丁版本。
本次发布聚焦于通用 E-R 图工作流的稳定性修复，解决 generic `dbdia-er` fallback 在多关系场景下可能导致正式 `release-build` 失败的问题。

## 本次修复

### 1. 修复 generic E-R fallback 的重复关系名问题

- `tools/core/figure_assets.py` 现在会为 generic `dbdia-er` E-R fallback 按 `target/current table` 组合生成稳定且唯一的关系标识。
- 当通用 SQL 结构中存在多个外键关系时，不再把所有关系都写成同一个 `关联` 名称。
- 因此，`dbdia` 在多关系模型中不再因为重复定义关系而终止渲染。

### 2. 补齐回归覆盖

- `tools/core/selftest.py` 新增多关系 generic E-R 样例与唯一性断言。
- 自测不再只覆盖单关系示例，而是明确验证“多关系 fallback 产出的关系名必须唯一”。
- 这使通用 E-R 渲染链的稳定性问题可以在 bundle 层更早暴露，而不是等到某个真实 workspace 构建时才失败。

## 对使用者的直接影响

- 不需要修改现有 workspace 配置，也不需要新增 `er_figure_specs`。
- 对依赖通用 E-R fallback 的工作区，`prepare-figures`、`release-build` 和 `release-verify` 在多关系模型下更稳定。
- 已采用显式 `er_figure_specs` 的 workspace 不受行为破坏，只会继承本次通用稳定性修复。
