# workflow_bundle v0.5.1 Release Notes

发布日期：2026-04-18

对应 tag：`v0.5.1`

## 概览

`v0.5.1` 是在 `v0.5.0` 基础上的补丁版本。
本次发布不引入新的 workflow 业务能力，重点修复版本查询结果中的一个边界问题：当仓库同时存在中文 milestone tag 与 semver tag 时，`latest_semver_tag` 可能错误返回预发布 tag。

## 本次修复

### 1. 修正 semver tag 选择逻辑

- `tools/core/bundle_version.py` 不再依赖混合 tag 列表上的 Git 排序结果来选取最新 semver tag。
- 当前实现会先过滤出符合 semver 规则的 tag，再按 semver 规则做本地比较。
- 在当前仓库结构下，`python3 workflow_bundle/tools/cli.py version` 现在会正确把最新 semver tag 识别为 `v0.5.1`，而不是继续停在 `v0.5.0-rc1`。

### 2. 保持既有发布历史不变

- `v0.5.0` 与 `v0.5.0-rc1` 都继续保留，不重写历史 tag。
- 这次修复通过补丁版本 `v0.5.1` 发布，而不是回改已经发布的正式版 `v0.5.0`。

## 对使用者的直接影响

- `python3 workflow_bundle/tools/cli.py version` 的 `latest_semver_tag` 输出现在与当前 semver 发布历史一致。
- 依赖该字段做发布判断、版本展示或机器摘要落盘的链路，不再受中文 milestone tag 混排影响。
- 已有 workflow 主流程、发布链和正文生成链不需要额外迁移。

## 回归结论

本次补丁版发布前已完成以下校验：

- `python3 -m py_compile tools/cli.py tools/core/bundle_version.py tools/core/release_summary.py tools/core/runtime_state.py tools/core/selftest.py`
- `bash workflow/scripts/check_bundle_sync.sh`
- `python3 tools/cli.py version`
- `python3 tools/cli.py selftest`
- `git diff --check`

## 已知边界

- 本次更新只修复 semver tag 识别问题，不新增新的 workflow 能力面。
- GitHub Release 页面仍可后续单独补充，但不影响仓库内 `VERSION`、tag 与 release note 的正式性。
