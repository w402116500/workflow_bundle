# workflow_bundle v0.6.0 Release Notes

发布日期：2026-04-18

对应 tag：`v0.6.0`

## 概览

`v0.6.0` 是基于 `v0.5.2` 的 minor 版本。
本次发布将“参考图规范抽取 -> guide 驱动 AI 技术图 -> 发布前阻断校验”收口为正式 workflow，同时把 PlantUML 与 `dbdia-er` 一并纳入稳定的公共图件合同。

## GitHub Release 页面文案

以下内容可直接粘贴到 GitHub Release 页面：

```md
workflow_bundle v0.6.0 是基于 v0.5.2 的 minor 版本。

本次更新把“先抽 guide，再生成 AI 技术图”的链路正式纳入公共 workflow，并补齐 PlantUML、版本治理与自测覆盖。

### 变更内容

- 新增 `prepare-reference-guides` 正式 CLI 命令。
- 新增 `reference_extraction`、`reference_guide_specs` 与 `ai_figure_specs.<fig>.reference_guides` 配置接口。
- 新增 `plantuml_figure_specs`，支持按图号声明 `.puml` 源驱动渲染，并写出 `.puml/.svg` sidecar。
- 非 ER 技术图现统一推荐先把参考资产冻结到 `docs/images/reference_guides_src/`，再执行 `prepare-reference-guides`。
- `version` 子命令现显式区分 `suggested_tag`、`tag_exists` 与 `tag_commit`，减少发版前 tag 语义误判。

### 影响

- 这是一个 minor 版本，因为新增了正式 CLI 子命令和新的 workspace 配置段。
- 已有 `reference_guides` 使用方式保持兼容，但推荐迁移到冻结 guide source 的写法。
- `ER` 图通常继续走 `er_figure_specs + dbdia-er`，不要求迁移到 AI guide。

### 验证

- `bash workflow_bundle/workflow/scripts/sync_root_compat.sh`
- `bash workflow_bundle/workflow/scripts/check_bundle_sync.sh`
- `python3 workflow_bundle/tools/cli.py version --json`
- `python3 workflow_bundle/tools/cli.py selftest`
- `git diff --check`
```

## 本次新增与收口

### 1. 参考图规范抽取成为正式阶段

- 新增 `tools/core/reference_guides.py`，负责：
  - 读取 `reference_extraction`
  - 解析 `reference_guide_specs`
  - 生成 `docs/images/reference_guides/*.json|*.md`
  - 校验 guide 是否缺失或过期
- 新增 `python3 workflow_bundle/tools/cli.py prepare-reference-guides` 作为正式入口。

### 2. AI 技术图开始消费 guide，而不是只靠参考图

- `prepare-ai-figures` 现支持 `ai_figure_specs.<fig>.reference_guides`。
- guide 摘要会进入 prompt manifest，发布前检查也会在 guide 缺失或过期时直接阻断。
- 文档和模板统一推荐对以下图类优先启用 guide：
  - `use_case`
  - `architecture`
  - `flowchart`
  - `function_structure`

### 3. 本地确定性图件链路继续保留并补稳

- `plantuml_figure_specs` 已正式纳入 workspace contract。
- `prepare-figures` 现统一校验 `plantuml` / `dbdia-er` sidecar 是否完整。
- vendored PlantUML 运行时会自动寻找 Java 11+，避免当前环境默认 Java 8 时直接失败。
- `ER` 图链路继续建议走 `er_figure_specs + dbdia-er`。

### 4. 版本治理语义更清楚

- `version` 命令现明确区分：
  - `suggested_tag`
  - `tag_exists`
  - `tag_commit`
- 这样可以避免“本地 `VERSION` 已修改，但 Git 仓库尚未创建 tag”时出现误判。

## 对使用者的直接影响

- 如果 workspace 仍把 `tmp_*`、外部绝对路径或 `docs/images/generated_ai/*` 直接当作 guide source，建议改为先复制到 `docs/images/reference_guides_src/`。
- 如果 workspace 已使用 `ER` 图的 `dbdia-er` 链路，本次不需要迁移。
- 如果准备引入 `PlantUML`，建议在 `.puml` 中显式声明 `!pragma layout smetana`，并确保本机有 Java 11+。

## 公共发版边界

- 本次 public release 不包含当前论文工作区的项目化章节默认值。
- 发布范围仅保留通用 workflow 能力、接口、模板、文档与回归。
