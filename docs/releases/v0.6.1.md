# workflow_bundle v0.6.1 Release Notes

发布日期：2026-04-18

对应 tag：`v0.6.1`

中文里程碑 tag：`工作区硬隔离保护-20260418`

## 概览

`v0.6.1` 是基于 `v0.6.0` 的补丁版本。
本次发布不新增公开 CLI 子命令，也不引入新的 workspace schema，而是把“bundle 示例配置只读化”和“禁止把项目运行状态写回公共仓库”收口为正式保护机制。

## GitHub Release 页面文案

以下内容可直接粘贴到 GitHub Release 页面：

```md
workflow_bundle v0.6.1 是基于 v0.6.0 的补丁版本。

本次更新把 bundle 示例配置与 in-repo workspace 的变更型命令正式阻断，避免新项目继续把 handoff、execution log 和工作区配置写回公共 workflow 仓库。

### 变更内容

- `current_workspace.json` 与 `current_project_manifest.json` 现在明确作为 bundle 示例实例使用。
- 对 bundle 示例配置或 `workspace_root` 落在 `workflow_bundle/` 内的配置，`set-active-workspace`、`extract`、`refresh-handoff`、`clear-lock` 及其他变更型命令会被直接拒绝。
- `resume`、handoff 与 `check-workspace` 现会显式显示 `workspace_mutation_safety`、reason codes 和迁移建议。
- `selftest` 新增 bundle 示例配置只读回归覆盖。
- `.gitignore` 新增 `docs/workflow/`，避免 bundle 根目录运行时状态产物误入版本库。

### 影响

- 这是一个 patch 版本，因为它主要修复 bundle 与真实项目 workspace 的边界管理问题，没有新增公开 CLI 子命令，也没有变更 workspace schema。
- 真实项目的推荐入口不变，仍然是先执行 `intake` 生成独立 workspace，再通过该 workspace 的 `workflow/configs/workspace.json` 继续后续命令。

### 验证

- `python3 workflow_bundle/tools/cli.py version`
- `python3 workflow_bundle/tools/cli.py version --json`
- `bash workflow_bundle/workflow/scripts/check_bundle_sync.sh`
- `python3 workflow_bundle/tools/cli.py selftest`
- `git diff --check`
```

## 本次修复内容

### 1. bundle 示例配置现在是正式只读入口

- `workflow/configs/current_workspace.json`
- `workflow/configs/current_project_manifest.json`

以上两个文件现在只用于 bundle 自带示例与只读查看，不再允许作为真实项目的变更入口。

### 2. in-repo workspace 会被正式阻断

- 如果某个配置解析出的 `workspace_root` 仍在 `workflow_bundle/` 仓库内部，CLI 会把它标为 `workspace_mutation_safety: blocked`。
- 这样可以防止：
  - handoff 写回 bundle 根目录
  - execution log 落到公共仓库
  - 新项目误把公共示例当成自己的正式 workspace

### 3. 诊断入口会明确提示迁移路径

- `resume`
- `check-workspace`
- handoff 渲染

这些入口现在不会沉默失败，而是会明确输出：

- `workspace_mutation_safety`
- `workspace_mutation_reason_codes`
- 推荐下一步：
  - `intake --project-root <project-root> --title <title> --out <workspace-dir>`
  - `set-active-workspace --config <workspace-dir>/workflow/configs/workspace.json`

## 对使用者的直接影响

- 如果你之前直接对 `workflow/configs/current_workspace.json` 运行变更型命令，现在会被拒绝。
- 正确做法是先为项目生成独立 workspace，再对项目 workspace 执行 `extract`、`scaffold`、`prepare-writing`、`release-build` 等命令。
- 如果只是想查看当前 bundle 示例配置状态，`resume` 之类的只读命令仍然可用。
